# Johari Cognitive Framework: 认知象限与模块定位

> **Status**: Design Framework (v1)
> **Purpose**: 为功能设计和规划提供认知框架
> **Last Updated**: 2024-12

---

## 1. 框架概述：约哈里窗 × 知识图谱

约哈里窗（Johari Window）是一个自我认知模型，我们将其应用于 Prism 的知识图谱设计：

```
                │  用户意识到               │  用户未意识到
────────────────┼─────────────────────────┼─────────────────────────
 系统能观察到   │  公开区 (Arena)          │  盲点区 (Blind Spot)
                │  "我知道 & 系统也知道"    │  "系统看到但我没注意"
────────────────┼─────────────────────────┼─────────────────────────
 系统不可观察   │  隐藏区 (Hidden)         │  未知区 (Unknown)
                │  "我知道但没告诉系统"    │  "双方都未发现"
```

**核心洞察**：传统知识管理工具只在"公开区"工作。Prism 的差异化在于同时服务所有四个象限。

## 2. 现有模块象限定位

### 2.1 象限全景图

```
                │  用户意识到               │  用户未意识到
────────────────┼─────────────────────────┼─────────────────────────
                │                         │
 系统能观察到   │  ┌─────────────────────┐ │  ┌─────────────────────┐
                │  │ Gardener 🌱         │ │  │ Serendipity 🎲      │
                │  │ Scout 🔭            │ │  │                     │
                │  │ DeepExplorer ⛏️     │ │  │ (认知闭环检测)       │
                │  │ Ripple 🌊           │ │  │                     │
                │  └─────────────────────┘ │  └─────────────────────┘
                │                         │
────────────────┼─────────────────────────┼─────────────────────────
                │                         │
 系统不可观察   │  ┌─────────────────────┐ │  ┌─────────────────────┐
                │  │ [需用户主动输入]    │ │  │ ❓ 空缺              │
                │  │                     │ │  │                     │
                │  │ (手动笔记/标注)      │ │  │ (最大机会区)         │
                │  └─────────────────────┘ │  └─────────────────────┘
                │                         │
```

### 2.2 模块详解

#### 🌱 Gardener — 公开区质量维护

```
位置: 公开区 (Arena)
方向: 内向 🔄
隐喻: 园丁（除草、修剪、防污染）

职责:
├── 重复检测 (Deduplicator)
├── 实体合并 (Merger)
├── 别名管理 (Aliases)
└── 审计追踪 (Audit Trail)

核心问题: "这两个实体是同一个吗？"
安全原则: "宁可重复，不可污染"
```

**认知价值**：确保公开区的信息**准确**，防止认知混乱。

---

#### 🔭 Scout — 公开区边界扩展

```
位置: 公开区 (Arena)
方向: 外向 →
隐喻: 侦察兵（巡逻、监控、情报收集）

职责:
├── LOD 调度 (Gravity-based)
├── 外部信息抓取 (Tavily/Web)
├── 实体资料更新 (Profiles)
└── 休眠唤醒 (Wake-up)

核心问题: "我关注的实体最近有什么变化？"
调度策略: 重力越高 → 巡逻越频繁
```

**认知价值**：让公开区保持**新鲜**，与外部世界同步。

---

#### ⛏️ DeepExplorer — 公开区深度挖掘

```
位置: 公开区 (Arena)
方向: 向下 ↓
隐喻: 矿工（深挖、提炼、发现底层价值）

职责:
├── 意图解析 (Intent Extraction)
├── 多头探索 (Multi-head)
├── 策略评估 (IDepthStrategy)
├── 迭代深化 (Forced Reflection)
└── 锚定输出 (Anchored to Intent)

核心问题: "这个想法的最深层洞察是什么？"
深度定义: 讽刺金字塔 (Irony Pyramid)
```

**认知价值**：把公开区的信息挖**深**，发现非显而易见的洞察。

---

#### 🌊 Ripple — 公开区关联传播

```
位置: 公开区 (Arena)
方向: 扩散 ◎
隐喻: 涟漪（一石激起千层浪）

职责:
├── 变化检测 (Change Detection)
├── 影响传播 (Propagation)
├── 相关实体更新 (Related Entities)
└── 热度传递 (Temperature Flow)

核心问题: "这个变化会影响什么？"
传播规则: 基于关系强度衰减
```

**认知价值**：让公开区保持**连贯**，变化自动传播。

---

#### 🎲 Serendipity — 盲点区揭示 (The Prism)

```
位置: 盲点区 (Blind Spot) → 公开区
方向: 折射 🌈
隐喻: 棱镜 (Prism) — 将白光分解为光谱

盲点不再是单一的黑洞，而是四维的认知缺失：

1. 🔵 逻辑盲点 (Structural)
   - 缺失: 系统动力学、因果链条
   - 症状: "我想当然..."
   - 解药: Causal Strategy (Mechanism/RootCause)

2. 🔴 情感盲点 (Emotional)
   - 缺失: 人性、共情、故事
   - 症状: "这只是冷冰冰的数据..."
   - 解药: Emotional Strategy (Empathy/Resonance)

3. 🟢 事实盲点 (Factual)
   - 缺失: 外部证据、权威数据
   - 症状: "我只看到了回声室..."
   - 解药: Evidence Strategy (Source/Data)

4. 🟣 反差盲点 (Irony)
   - 缺失: 矛盾、荒谬、宿命
   - 症状: "我没看到这件事的可笑之处..."
   - 解药: Irony Strategy (Surprise/Satire)
```

**认知价值**：Prism 引擎的作用就是根据用户当前状态，选择正确的光谱去**照亮**盲点。只有当这四束光汇聚时，真正的 "Serendipity" 才会发生。

## 3. 象限空白分析

### 3.1 隐藏区 (Hidden) — 用户知道但系统不知道

**当前状态**: 被动依赖用户输入

**问题**:
- 用户有很多想法没有记录
- 用户的真实偏好没有显式表达
- 用户的背景知识系统无法感知

**潜在机制**:

| 机制 | 描述 | 实现难度 |
|------|------|----------|
| 反馈学习 | 从用户行为（点击、停留、忽略）推断偏好 | 中 |
| 主动询问 | 系统主动问"你对这个怎么看？" | 低 |
| 情绪推断 | 从写作风格推断用户状态 | 高 |
| 隐式标注 | 用户选择/高亮行为作为隐式信号 | 中 |

---

### 3.2 未知区 (Unknown) — 双方都不知道

**当前状态**: 完全空白 ❓

**问题**:
- 信息茧房效应
- 无法发现"不知道自己不知道的"
- 潜能未被激发

**潜在机制**:

| 机制 | 描述 | 实现思路 |
|------|------|----------|
| **随机播种** | 定期注入与用户图谱无关的随机实体 | 全局热点采样 |
| **反向推荐** | 找与用户兴趣"负相关"的内容 | Anti-Gravity 算法 |
| **邻居窥探** | 匿名查看相似用户关注什么 | 联邦学习/差分隐私 |
| **视角切换** | "如果你是 [另一个领域的人]，你会怎么看？" | LLM 角色扮演 |
| **时间穿越** | 重新审视历史被忽略的信息 | 休眠实体复活 |
| **对立面探索** | 主动搜索用户观点的反面 | Adversarial 方向 |

## 4. 模块协作流程

### 4.1 标准流程：新信息进入

```
[外部信息] 
    │
    ▼
┌──────────┐     ┌──────────┐     ┌──────────┐
│  Ingest  │────▶│  Ripple  │────▶│ Gardener │
│  (摄入)   │     │  (传播)   │     │  (清理)   │
└──────────┘     └──────────┘     └──────────┘
                      │
                      ▼
              ┌──────────────┐
              │ Serendipity  │
              │ (闭环检测)    │
              └──────────────┘
```

### 4.2 用户查询流程

```
[用户提问]
    │
    ▼
┌──────────────┐
│ DeepExplorer │
│ (意图解析)    │
└──────────────┘
    │
    ├──▶ [本地图谱] ◀── Scout (补充外部信息)
    │
    ▼
┌──────────────┐
│ 深度挖掘循环  │
│ (多轮迭代)    │
└──────────────┘
    │
    ▼
[锚定输出] + [Serendipity 埋入]
```

### 4.3 后台维护流程

```
[定时触发]
    │
    ├──▶ Scout Tick (LOD 巡逻)
    │
    ├──▶ Gardener Scan (重复检测)
    │
    └──▶ Physics Update (重力计算)
```

## 5. 未来规划：填补空白象限

### 5.1 Phase 1: 增强隐藏区感知

**目标**: 减少用户显式输入负担，增强隐式学习

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 行为追踪 | 记录点击/停留/忽略模式 | P1 |
| 反馈循环 | "这个有用吗？" 微交互 | P1 |
| 写作分析 | 从笔记风格推断兴趣变化 | P2 |

### 5.2 Phase 2: 探索未知区

**目标**: 打破信息茧房，发现未知潜能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| RandomSeed Agent | 每周注入 1-3 个"随机种子"实体 | P1 |
| AntiGravity Explorer | 主动搜索低重力区域的有趣内容 | P2 |
| Perspective Shift | LLM 扮演不同角色审视用户问题 | P2 |
| Dormant Resurrection | 定期复活休眠实体，检查新价值 | P3 |

### 5.3 Phase 3: 跨象限协同

**目标**: 让四个象限形成闭环

```
未知区 ──RandomSeed──▶ 公开区
                          │
                      Serendipity
                          │
                          ▼
盲点区 ◀─────────────── 揭示
    │
    │ 用户发现
    ▼
公开区 ◀── 用户确认 ── 隐藏区
```

## 6. 设计原则

### 6.1 四象限平衡原则

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   不要只优化公开区。                                 │
│   真正的认知增强来自于扩展整个窗口的面积。            │
│                                                     │
│   公开区太大 → 信息过载                              │
│   盲点区太大 → 自以为是                              │
│   隐藏区太大 → 系统不懂你                            │
│   未知区太大 → 潜能未释放                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 6.2 渐进揭示原则

```
不要直接告诉用户他的盲点。
让他自己发现，这样印象更深。

Serendipity 的设计哲学：
"埋下种子，等待开花"
```

### 6.3 安全边界原则

```
未知区探索有风险：
- 可能推送用户完全不感兴趣的内容 → 骚扰
- 可能挑战用户深层信念 → 不适
- 可能暴露用户不想知道的真相 → 伤害

安全策略：
1. 随机种子可被快速忽略，不影响核心体验
2. 未知区探索是 opt-in 功能
3. 保留"不再推荐类似内容"机制
```

## 7. 指标体系

### 7.1 象限健康度指标

| 象限 | 指标 | 健康范围 |
|------|------|----------|
| 公开区 | 实体活跃率 (LOD 0-2 / Total) | 10-30% |
| 公开区 | 去重候选数 | < 5% of entities |
| 盲点区 | 闭环触发率 | 1-5 / month |
| 隐藏区 | 用户输入频率 | 观察 |
| 未知区 | 随机种子接受率 | > 10% |

### 7.2 认知扩展指标

```
窗口扩展速度 = 新实体引入率 + 新关联发现率
认知深化速度 = DeepExplorer 平均深度 / 查询
惊喜指数 = Serendipity 闭环强度加权平均
```

## 9. 附录：约哈里窗原始模型

约哈里窗由 Joseph Luft 和 Harry Ingham 于 1955 年提出，原用于人际沟通和自我认知。

**原始四象限**:
1. **Arena (公开区)**: 自己和他人都知道的特质
2. **Blind Spot (盲点区)**: 他人知道但自己不知道的特质
3. **Façade (隐藏区)**: 自己知道但他人不知道的特质
4. **Unknown (未知区)**: 自己和他人都不知道的特质

**在 Prism 中的映射**:
- "自己" → 用户
- "他人" → 系统 (AI + 知识图谱)
- "特质" → 实体、关系、洞察

**核心转换**: 从人际沟通模型到人机认知协作模型。
