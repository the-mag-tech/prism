# User Storytelling: 用户向的叙事设计

> **Purpose**: 将内部技术概念转化为用户能理解的故事
> **Audience**: 产品设计、文案、Onboarding
> **Related**: [ORIGIN-RELATIVE-EMBEDDING.md](file:///Users/j.z/code/fulmail/apps/prism-server/docs/ORIGIN-RELATIVE-EMBEDDING.md)

---

## 1. 核心原则

> **用户不需要理解物理公式，只需要感受到系统"懂"他们。**

---

## 2. 术语翻译表

| 内部术语 | 用户语言 | 含义 |
|---------|---------|------|
| Gravity | "浮力" / "吸引力" | 这个东西对你有多重要（越重要越靠近水面/中心） |
| Origin | "你的位置" | 你现在在泡什么汤 |
| Origin Drift | "游动" | 你的兴趣在移动 |
| Field | "汤屋" | 你所有的人、事、想法所在的这个空间 |
| Entity | "气泡" / "物件" | 一个具体的节点 |
| Relation | "水流" | 两个事物之间的连结 |
| Path Association | "习惯路径" | 你经常顺着游的方向 |
| Embedding | (不暴露) | 内部技术细节 |
| Similarity | "共鸣" | 两个东西有多像 |
| Irony Atom | "惊喜气泡" | 和你预期不一样的东西（突然冒出来的） |

## 3. 用户常见问题 & 故事化回答

### Q: 为什么这个人出现在我眼前？

> *"因为水流把他推过来了。可能是你们最近有互动，或者他在你的关注圈里。"*

**不要说**: "因为他的 Gravity 是 0.85"

---

### Q: 为什么今天看到的东西和昨天不一样？

> *"因为你在汤屋里的位置变了。昨天你在泡‘设计’汤，今天你游到了‘AI’区。"*

**不要说**: "因为 Origin 漂移了 0.3 个单位"

---

### Q: 为什么 AI 觉得这个想法有意思？

> *"就像泡澡时突然冒出的灵感气泡。它和你平时的想法不太一样，所以特别显眼。"*

**不要说**: "因为 Irony Atom 检测到 Level 4"

---

### Q: 为什么看到 A 就会想起 B？

> *"因为水流习惯了这么走。你经常把它们放在一起看。"*

**不要说**: "因为 Path Association 权重是 0.7"

---

### Q: 为什么某些人/事消失了？

> *"可能是因为你最近没怎么理他们，他们就慢慢沉下去了，或者飘远了。"*

**不要说**: "因为 Temperature 衰减到了 0.1"

---

## 4. Onboarding 叙事

### 4.1 首次使用

> **"欢迎来到汤屋。"**
> 
> 这里是你的思维澡堂。
> 
> 对你重要的人和事，会像气泡一样浮在你身边。
> 
> 你越关注什么，它就离你越近。
> 
> 不重要的杂念会慢慢沉底。

### 4.2 发现新内容

> **"有个新气泡冒出来了。"**
> 
> 它可能顺着你关注的 X 飘过来的。
> 
> 或者是给你的惊喜 —— 一个意想不到的灵感。

### 4.3 关注点变化

> **"你在游动。"**
> 
> 上周你在想 A，这周你游到了 B。
> 
> 这很正常。汤屋的水流会跟着你动。

## 5. UI Tooltip 文案

### 5.1 实体卡片

| 位置 | 文案 |
|------|------|
| 高亮卡片 | "最近一直浮在你身边" |
| 新出现的卡片 | "新冒出来的气泡" |
| 带火花标记 | "一个惊喜的发现" |
| 带时钟标记 | "正在靠近" |

### 5.2 Feed 区域

| 状态 | 文案 |
|------|------|
| 空状态 | "水面很平静。扔点东西进来看看涟漪。" |
| 加载中 | "正在加热..." |
| 结果刷新 | "水流方向变了" |

### 5.3 搜索/筛选

| 位置 | 文案 |
|------|------|
| Lens 切换 | "换个池子泡泡" |
| 时间筛选 | "看看之前的池水里有什么" |

## 6. 隐喻体系

系统使用的隐喻应该是**一致的**。推荐使用：

| 隐喻 | 适用场景 |
|------|---------|
| **汤屋 (Bathhouse)** | 用户的整体信息空间 |
| **位置 (Spot)** | 用户当前的注意力中心 |
| **上浮/下沉 (Rise/Sink)** | 重要性的变化 |
| **流动 (Flow)** | 系统随时间变化 |
| **气泡 (Bubble)** | 灵感或实体 |

避免使用：
- 物理术语 (Gravity, Field, Force) - *除了内部算法保留 Gravity*
- 量子术语 (Quantum, Entanglement)
- 硬核科幻术语

---

## 7. 情感设计

系统应该让用户感到：

| 情感 | 如何实现 |
|------|---------|
| **被理解** | "这个东西确实和我有关" |
| **惊喜** | "我没想到还有这个" |
| **掌控** | "我可以调整我的关注点" |
| **成长** | "我的世界在变得更丰富" |

避免让用户感到：
- 被监控 ("它怎么知道这么多？")
- 困惑 ("为什么是这个？")
- 失控 ("为什么它消失了？")
